GIT 与 svn 主要区别

  • 存储方式不一样

    GIT把内容按元数据方式存储类似k/v数据库，而SVN是按文件(新版svn已改成元数据存储)
    git 存储过程演示:

  • 使用方式不一样
    从本地把文件推送远程服务，SVN只需要commint 而GIT需要pull 、 add、commint、push 四个步骤

  • 管理模式不一样
    git 是一个分布式的版本管理系统，而要SVN是一个远程集中式的管理系统

Git实战

这是今天的重头戏，花了我好长时间才搭建好私服，我来讲讲大致的过程。

  1. 在腾讯云centOS 7上安装docker

  2. 下载gogs的docker镜像
    docker pull gogs/gogs

  3. 创建gogs存储的目录
    mkdir -p /var/gogs

  4. 运行镜像, 产生容器
    docker run --name=gogs -p 10022:22 -p 10080:3000 -v /var/gogs:/data gogs/gogs

  5. 然后就可以通过浏览器访问服务器公网ip地址:10080进去配置页面

  6. 在新搭建的站点上, 创建新仓库

  7. 通过http方式获取仓库
    在命令行通过http方式获取新创建的仓库并克隆到本地新建文件夹，创建文件后进行pull，add，commit，push操作，操作完成后就可在仓库里看到提交的文件。
